# ペルソナ定義

## 概要

本ドキュメントでは、AIコードレビュー学習サービスのターゲットユーザーを3つのペルソナとして定義する。各ペルソナは実際のユーザーインタビューや市場調査に基づく仮説であり、今後の検証を通じて更新される。

## ペルソナ詳細

### ペルソナ1：田中 健太（34歳）★★★ プライマリー

#### 基本プロフィール

| 項目 | 内容 |
|------|------|
| 年齢 | 34歳 |
| 職種 | メーカー勤務・DX推進室 |
| 家族構成 | 妻、子供1人（3歳） |
| 居住地 | 東京都郊外 |
| 年収 | 550万円 |
| 学習可能時間 | 平日30分〜1時間（通勤時間・昼休み）、休日2時間 |

#### 背景・経歴

- 文系大学卒業後、営業職として入社
- 3年前に社内公募でDX推進室へ異動
- Udemyや書籍でPython基礎を独学（約6ヶ月）
- 現在はExcel VBAとPythonで簡単な業務自動化スクリプトを作成

#### 現在の課題

1. **コードの品質に自信がない** - 動くコードは書けるが、「このやり方で合っているのか」が分からない
2. **レビューしてくれる人がいない** - チーム内にエンジニアがおらず、フィードバックを得る機会がない
3. **セキュリティリスクが心配** - 社内データを扱うスクリプトを書く際、脆弱性がないか不安
4. **学習時間の確保が難しい** - 子育てと仕事の両立で、まとまった学習時間が取れない

#### 学習動機

- 社内で「DX人材」として評価されたい
- 業務自動化の成果を出して昇進・昇給につなげたい
- 将来的にはチームメンバーにも指導できるレベルになりたい

#### 期待する成果

- **短期（3ヶ月）**：自分のコードの問題点を把握し、改善できる
- **中期（6ヶ月）**：セキュリティ・パフォーマンスを意識したコードが書ける
- **長期（1年）**：社内の他メンバーにコードレビューができるレベルになる

#### 利用シーン

- 通勤電車内でスマホからコードを入力し、レビュー結果を確認
- 昼休みにPCでコードを貼り付け、改善点を学習
- 週末に1週間分のコードをまとめてレビューし、学習ノートを作成

#### 技術レベル

| スキル | レベル（1-5） |
|--------|-------------|
| Python | 2（基礎文法は理解） |
| JavaScript | 1（ほぼ未経験） |
| Git | 2（基本操作のみ） |
| SQL | 2（SELECT文程度） |

- **長期（1年）**：RPAプロジェクトで中心的な役割を担える

#### 利用シーン

- 大学卒業後、IT企業に営業職として入社
- エンジニアと協働する中でプログラミングに興味を持つ
- Progateでプログラミングの基礎を学習（HTML/CSS/JavaScript）

- **短期（3ヶ月）**：転職面接で評価されるレベルのコードが書ける
- **中期（6ヶ月）**：エンジニアとして転職成功
- **長期（1年）**：実務で即戦力として活躍できる

#### 利用シーン

- 平日夜の学習時間に、書いたコードをすぐにレビュー
- 週末のポートフォリオ開発時に、重要なコードを重点的にレビュー
- 転職面接前に、提出コードの最終チェック

## ペルソナ間の共通点

1. **時間制約** - 全員が仕事や家庭との両立で学習時間が限られている
2. **フィードバック不足** - 独学でコードの良し悪しを判断してくれる人がいない
3. **実践志向** - 学習自体が目的ではなく、業務やキャリアに活かしたい
4. **品質への不安** - 「動くけど、これでいいのか？」という漠然とした不安を抱えている
