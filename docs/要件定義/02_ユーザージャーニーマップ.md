# ユーザージャーニーマップ

## 概要

本ドキュメントでは、AIコードレビュー学習サービスにおけるユーザーの体験を、認知から習慣化までの4フェーズで可視化する。プライマリーペルソナ「田中健太」を基準に作成し、他ペルソナへの適用時の差異も補足する。

---

## ジャーニー全体像

```
認知 → 登録 → 初回利用 → 習慣化
(1週間)  (10分)  (30分)    (継続)
```

---

## フェーズ1：認知

### 概要

| 項目 | 内容 |
|------|------|
| 期間 | サービス発見〜登録検討まで（1日〜1週間） |
| ゴール | サービスの価値を理解し、「試してみたい」と思う |

### タッチポイント

| チャネル | 具体的な接点 |
|---------|-------------|
| SNS | X(Twitter)でのユーザー投稿・広告 |
| 検索 | 「Python コードレビュー」「AIコードチェック」等で検索 |
| 口コミ | 社内同僚・オンラインコミュニティでの紹介 |
| 記事 | Qiita・Zennの紹介記事、比較記事 |

### ユーザーの行動

1. SNSまたは検索でサービスを発見
2. ランディングページにアクセス
3. サービス概要・機能を確認
4. 料金プラン・無料枠を確認
5. レビュー・口コミを探す

### ユーザーの思考・感情

| 感情 | 思考 |
|------|------|
| 😮 興味 | 「AIがコードレビューしてくれるの？便利そう」 |
| 🤔 疑問 | 「本当に役に立つレビューがもらえるの？」 |
| 😟 不安 | 「自分のコードを外部サービスに送っても大丈夫？」 |
| 💰 懸念 | 「無料で試せるなら試してみたい」 |

### ペインポイント

1. **サービスの信頼性が不明** - 実際のレビュー品質が分からない
2. **セキュリティへの懸念** - コードを外部に送信することへの抵抗
3. **類似サービスとの違いが不明** - GitHub Copilot等との差別化が見えない
4. **効果の実感が難しい** - 使う前に価値が分からない

### 対応施策（MVP範囲）

| ペインポイント | 施策 |
|---------------|------|
| 信頼性不明 | デモ機能（サンプルコードでレビュー体験） |
| セキュリティ懸念 | セキュリティポリシーの明示、データ保持期間の説明 |
| 差別化不明 | 「学習特化」の価値訴求、比較表の掲載 |

## フェーズ2：登録

### 概要

| 項目 | 内容 |
|------|------|
| 期間 | 登録開始〜完了まで（5〜10分） |
| ゴール | スムーズにアカウント作成を完了し、すぐに使い始められる |

### タッチポイント

| 画面 | 内容 |
|------|------|
| 登録フォーム | メールアドレス・パスワード入力 |
| メール認証 | 認証メール受信・リンククリック |
| 初期設定 | プロフィール・学習目標設定（スキップ可能） |

### ユーザーの行動

1. 「無料で始める」ボタンをクリック
2. メールアドレス・パスワードを入力
3. 利用規約に同意
4. 認証メールを確認し、リンクをクリック
5. （任意）初期プロフィールを設定

### ユーザーの思考・感情

| 感情 | 思考 |
|------|------|
| ⏱️ 焦り | 「早く使ってみたい、登録は簡単に済ませたい」 |
| 😤 面倒 | 「入力項目が多いと離脱したくなる」 |
| ✅ 安心 | 「メール認証があるから、ちゃんとしたサービスだな」 |

### ペインポイント

1. **登録項目が多すぎる** - 入力が面倒で離脱
2. **認証メールが届かない** - 迷惑メールフォルダ行き
3. **パスワード要件が厳しい** - 何度もやり直しが必要
4. **初期設定の意図が不明** - なぜ設定が必要か分からない

### 対応施策（MVP範囲）

| ペインポイント | 施策 |
|---------------|------|
| 登録項目多い | 最小限の入力（メール・パスワードのみ） |
| メール届かない | 再送機能、迷惑メールフォルダ確認の案内 |
| パスワード要件 | リアルタイムバリデーション表示 |
| 初期設定不明 | 初期設定はスキップ可能にする |

### 登録フロー（MVP）

```
[メールアドレス入力] → [パスワード入力] → [利用規約同意]
        ↓
[認証メール送信] → [メール内リンククリック] → [登録完了]
        ↓
[ダッシュボード表示] → [初回利用ガイド表示]
```

## フェーズ3：初回利用

### 概要

| 項目 | 内容 |
|------|------|
| 期間 | 登録完了〜最初のレビュー体験まで（10〜30分） |
| ゴール | 「このサービスは使える！」というアハモーメントを体験 |

### タッチポイント

| 画面 | 内容 |
|------|------|
| ダッシュボード | 利用ガイド・クイックスタート表示 |
| コード入力画面 | コードエディタ・言語選択・レビュー実行 |
| レビュー結果画面 | AIからのフィードバック表示 |

### ユーザーの行動

1. ダッシュボードで操作方法を確認
2. 「新規レビュー」ボタンをクリック
3. プログラミング言語を選択
4. コードをペースト（または直接入力）
5. 「レビューを実行」ボタンをクリック
6. レビュー結果を確認
7. 指摘事項の詳細を読む
8. （任意）コードを修正して再レビュー

### ユーザーの思考・感情

| 感情 | 思考 |
|------|------|
| 😃 期待 | 「どんなレビューが返ってくるんだろう」 |
| 🤔 戸惑い | 「どのコードを試せばいいかな」 |
| 😲 驚き | 「こんな指摘もらえるんだ！」（アハモーメント） |
| 📚 学び | 「なるほど、こう書けばいいのか」 |
| 😊 満足 | 「これは続けて使いたい」 |

### ペインポイント

1. **何をすればいいか分からない** - 最初の一歩が踏み出せない
2. **レビュー対象のコードがない** - 手元にコードがない
3. **レビュー結果が理解できない** - 専門用語が多い
4. **レビュー時間が長い** - 待ち時間でストレス
5. **指摘が多すぎて圧倒される** - 何から手をつければいいか分からない

### 対応施策（MVP範囲）

| ペインポイント | 施策 |
|---------------|------|
| 何をすればいいか不明 | 初回利用ガイド（ステップバイステップ） |
| コードがない | サンプルコードボタン（ワンクリックで挿入） |
| 結果が理解できない | 初心者向け解説モード、用語説明ツールチップ |
| 待ち時間 | ローディングアニメーション、進捗表示 |
| 指摘多すぎ | 優先度順に表示、「まず直すべき点」を強調 |

### 初回利用フロー（MVP）

```
[ダッシュボード] → [初回ガイドモーダル表示]
        ↓
[コード入力画面] → [サンプルコード挿入可能]
        ↓
[言語選択] → [Python/JavaScript/TypeScript]
        ↓
[レビュー実行] → [ローディング（3-5秒）]
        ↓
[レビュー結果表示] → [優先度順にソート]
        ↓
[詳細解説表示] → [初心者向け説明]
```

## フェーズ4：習慣化

### 概要

| 項目 | 内容 |
|------|------|
| 期間 | 初回利用後〜継続利用（2週間〜） |
| ゴール | 週に2回以上の定期的な利用、学習習慣の定着 |

### タッチポイント

| チャネル | 内容 |
|---------|------|
| メール | 学習リマインダー、週間レポート |
| ダッシュボード | 学習進捗、過去レビュー履歴 |
| レビュー画面 | 継続利用で得られる機能開放 |

### ユーザーの行動

1. 定期的にサービスにアクセス
2. 業務で書いたコードをレビューに投稿
3. 過去のレビュー履歴を振り返る
4. 学習進捗を確認
5. 新しい観点でのレビューに挑戦

### ユーザーの思考・感情

| 感情 | 思考 |
|------|------|
| 📈 成長実感 | 「前より指摘が減ってきた！」 |
| 🎯 目標意識 | 「今週も3回はレビューしよう」 |
| 🔄 習慣化 | 「コード書いたらレビューに出すのが当たり前」 |
| 😔 マンネリ | 「いつも同じような指摘だな...」 |
| 🚀 向上心 | 「もっと高度な観点でレビューしてほしい」 |

### ペインポイント

1. **モチベーション低下** - 新鮮味がなくなる
2. **成長実感の欠如** - 上達しているか分からない
3. **忙しくて忘れる** - サービスの存在を忘れる
4. **機能の物足りなさ** - より高度な機能がほしい

### 対応施策（MVP範囲）

| ペインポイント | 施策 |
|---------------|------|
| モチベーション低下 | 学習ストリーク表示、小さな達成感の演出 |
| 成長実感の欠如 | 過去との比較レポート（指摘数の推移等） |
| 忙しくて忘れる | メールリマインダー（週1回、オプトアウト可能） |

### 習慣化フロー（MVP）

```
[週次リマインダーメール] ← オプトイン設定
        ↓
[ダッシュボード] → [学習ストリーク表示]
        ↓
[過去レビュー履歴] → [指摘傾向の可視化]
        ↓
[レビュー実行] → [継続的な学習]
```

## ペルソナ別ジャーニー差異

### 佐藤美咲（セカンダリー）の場合

| フェーズ | 特徴的な差異 |
|---------|-------------|
| 認知 | 口コミ・社内紹介が主な接点、SNSは少ない |
| 登録 | Google認証への抵抗感（セキュリティ懸念） |
| 初回利用 | サンプルコード活用率が高い、解説をじっくり読む |
| 習慣化 | 週末中心の利用、平日は短時間のみ |

### 山本翔平（ターシャリー）の場合

| フェーズ | 特徴的な差異 |
|---------|-------------|
| 認知 | 技術ブログ・比較記事からの流入が多い |
| 登録 | スピーディーに完了、設定もしっかり入力 |
| 初回利用 | 自分のポートフォリオコードを即座に投稿 |
| 習慣化 | 高頻度利用（週4-5回）、機能要望が多い |
