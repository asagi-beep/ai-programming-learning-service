# ユーザーニーズ洗い出し

## 概要

本ドキュメントでは、AIコードレビュー学習サービスのターゲットユーザーが持つニーズを、機能的・感情的・文脈的の3カテゴリに分類し、優先度を付与する。ペルソナ定義およびユーザージャーニーマップから導出した内容である。

---

## ニーズ分類体系

| カテゴリ | 定義 | 例 |
|---------|------|-----|
| 機能的ニーズ | サービスが提供すべき具体的な機能・能力 | 「コードの問題点を指摘してほしい」 |
| 感情的ニーズ | ユーザーが体験したい感情・心理状態 | 「成長を実感したい」 |
| 文脈的ニーズ | 特定の状況・環境で生じるニーズ | 「通勤中にスマホで使いたい」 |

## ニーズと技術制約の整合性

### Claude Haiku 4.5の制約考慮

| 制約 | 影響するニーズ | 対応方針 |
|------|---------------|---------|
| トークン制限（約200K） | F-08 複数ファイル | 単一ファイル・1000行以内に制限 |
| レイテンシー（1-3秒） | C-01 短時間 | 適切な期待値設定（3-5秒） |
| コスト（入出力トークン課金） | F-07 修正コード例 | 出力トークンの最適化 |

### MongoDB / Next.js の制約考慮

| 制約 | 影響するニーズ | 対応方針 |
|------|---------------|---------|
| MongoDBドキュメントサイズ | F-09 履歴確認 | コード・結果の圧縮保存 |
| Next.js API Routes タイムアウト | C-01 短時間 | Streaming対応検討 |
